## P0-1：BSC fourAdapter 接入真实链上签名与真实首买

- 把 `adapters/fourAdapter.ts` 从 stub 补成真实可执行版本
- 使用 `ethers.js`
- 使用 BSC RPC
- 使用 `bsc_deployer` 和 `bsc_ops_buy`
- 首买只能由单 `ops_buy` 钱包执行一次
- 返回真实结构化结果

必须实现：
1. `validate(payload)` 校验：
   - token_name
   - token_symbol
   - description
   - logo_url
   - deployer wallet
   - BSC RPC 配置
2. `launch(payload)` 返回：
   - `success`
   - `deployTxHash`

   - `platform`
   - `platformUrl`
   - `raw`
3. `opsBuy(payload)` 返回：
   - `success`
   - `buyTxHash`
   - `amount`
   - `walletLabel`

5. 所有失败写入 `audit_logs`
6. 私钥绝不能出现在日志里

要求：
- 如果当前 Four 真 API 尚未完全可用，至少做到：
  - 真实 BSC 钱包签名能力
  - 真实链上原生转账 / 调用能力
  - 清晰的适配层结构
- 保留 stub fallback，但必须以 feature flag 控制，且默认优先真实逻辑

验收标准：
- 能在本地或测试环境跑通一次真实签名流程
- `launch_jobs` 能写入真实 `deployTxHash`
- `opsBuy` 能写入真实 `buyTxHash`
- TG 通知里能展示真实哈希或明确 fallback 状态
