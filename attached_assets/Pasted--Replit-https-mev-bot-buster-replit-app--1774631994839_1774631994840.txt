### 1. 先审计，后修改

- 哪些部分可以复用
- 哪些部分需要新增
- 哪些部分不要动

然后再开始改代码。

### 2. 必须是“最小改动”
不要大重构，不要大改现有 anti-mev / bot 系统。
优先：
- 复用现有 Flask / FastAPI / scheduler / db / util / wallet / tg 代码
- 在现有基础上新增模块
- 如果已有 models.py，就在原模型文件里追加新表
- 如果已有 scheduler，就在现有 scheduler 中挂新任务
- 如果已有 Telegram 模块，就在原模块里扩展审批功能
- 如果已有 web 后台，就只加菜单和页面，不要整体重写 UI

### 3. 明确限制
必须遵守：
- 只做 **单公开运营钱包首买**
- 每条链只允许 3 个固定用途钱包：
  - deployer
  - treasury
  - ops_buy
- 所有关键动作必须写入 audit_logs
- 所有 AI 输出必须严格 JSON
- 所有失败都要有明确错误

明确不要做：
- 不实现多钱包捆绑买入
- 不实现 bundled tx 风格同区块成交
- 不实现刷量、假活跃、假持仓分布
- 不动态批量生成钱包
- 不使用 eval 解析模型输出
